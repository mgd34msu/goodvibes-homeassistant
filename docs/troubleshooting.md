# Troubleshooting

Start with the daemon health checks the config flow uses:

- `GET /status`
- `GET /api/homeassistant/health`
- `POST /api/channels/actions/homeassistant/homeassistant-manifest`
- `GET /api/homeassistant/home-graph/status` when Home Graph is enabled

If the daemon SDK was upgraded or restarted while Home Assistant was running, restart Home Assistant after the daemon reports healthy. That clears stale daemon clients and stale Home Graph state inside the integration runtime.

## Common Problems

| Symptom | Likely cause | Fix |
| --- | --- | --- |
| Config flow cannot connect to daemon | Wrong daemon URL, daemon not listening, Home Assistant cannot reach host, or Home Assistant surface disabled | Check `GET /status`, enable `featureFlags.homeassistant-surface`, set `surfaces.homeassistant.enabled: true`, and use a URL reachable from Home Assistant. |
| Config flow fails on Home Assistant health | Daemon Home Assistant surface is not configured or cannot use its Home Assistant access token | Check `surfaces.homeassistant.instanceUrl`, `surfaces.homeassistant.accessToken`, and `GET /api/homeassistant/health`. |
| Auth fails or returns `401` | Wrong daemon bearer token, token not sent, or a `goodvibes://` daemon secret was entered where Home Assistant needs the token value | Enter the companion/operator token generated by the GoodVibes pairing flow. `goodvibes://...` works inside daemon config, but Home Assistant needs a bearer token it can send. |
| Assist returns daemon `500` | Daemon SDK is too old, daemon conversation endpoint is unhealthy, or the daemon is stuck on a stale runtime state | Update the daemon to SDK `0.34.0` or newer, check `POST /api/homeassistant/conversation`, restart the daemon, then restart Home Assistant. |
| Home Graph is unavailable | Home Graph disabled in the integration, daemon route missing, daemon auth lacks admin privileges, or stale client after daemon restart | Enable Home Graph in the config entry, check `GET /api/homeassistant/home-graph/status`, confirm the token has access to mutating routes, and restart Home Assistant after daemon updates. |
| Uploads time out or fail for large files | Home Assistant, reverse proxy, or daemon artifact limits/timeouts are too low | Use the sidebar multipart upload path, avoid base64 JSON uploads, check proxy body limits and timeouts, and align daemon `storage.artifacts.maxBytes`. SDK `0.34.0` defaults daemon artifacts to `512 MiB`. |
| Ask, Pages, or Map look stale after SDK restart | Home Assistant still has an old daemon client or Home Graph snapshot | Restart Home Assistant, run `goodvibes.sync_home_graph`, then retry Ask/Pages/Map. |
| Missing manuals or source questions remain visible after linking evidence | Old issue records, old extraction, or stale generated pages remain in the daemon-owned space | Run `goodvibes.home_graph_reindex`, refresh generated pages, then reload Review/Pages. If old bad data remains, use reset preview and rebuild the space. |
| Sources are present but not linked to the right object | Source was ingested without a target hint, auto-link confidence was low, or the wrong graph object was selected | Use Review/Link in the sidebar or `goodvibes.link_knowledge` with `target_kind`, `target_id`, and `relation`. |
| Need to recover from bad ingest/link/page data | The selected Home Graph space is contaminated by historical data | Export for diagnosis, run `goodvibes.home_graph_reset` with `dry_run: true`, then destructive reset with `confirm: RESET`, sync, reingest/relink, reindex, and retest. |

## Credential Checks

The daemon config needs the Home Assistant long-lived access token:

```json
{
  "surfaces": {
    "homeassistant": {
      "instanceUrl": "http://homeassistant.local:8123",
      "accessToken": "goodvibes://...",
      "webhookSecret": "..."
    }
  }
}
```

The Home Assistant integration config needs:

- Daemon URL
- Daemon bearer token or session token
- Home Assistant webhook secret
- Event type, usually `goodvibes_message`

The webhook secret and Home Assistant long-lived access token are not substitutes for the daemon bearer token. The first request from Home Assistant to the daemon must include an actual bearer token in the `Authorization` header.

## Assist Checks

Assist uses:

```http
POST /api/homeassistant/conversation
Authorization: Bearer <daemon operator token>
```

The integration expects the daemon to return `assistant.speechText` or `assistant.text`. It does not wait for `/webhook/homeassistant` events for spoken Assist output.

If Assist returns an error:

1. Check `GET /status`.
2. Check `GET /api/homeassistant/health`.
3. Check the daemon logs around `/api/homeassistant/conversation`.
4. Restart the daemon.
5. Restart Home Assistant after the daemon is healthy.

Use `goodvibes.cancel` with `session_id` or `message_id` when an active Assist turn needs to be cancelled.

## Upload Checks

Use the `GoodVibes Home` sidebar for normal browser file uploads. That path uses multipart upload to Home Assistant, then forwards the temporary file to the daemon.

Avoid JSON base64 uploads for large PDFs, manuals, receipts, or photos. They are larger, slower, and more likely to hit Home Assistant or proxy limits.

For large files, check:

- Home Assistant request body limit
- Reverse proxy body size limit
- Reverse proxy read/write timeout
- Daemon `storage.artifacts.maxBytes`
- Daemon extraction/indexing logs

URL, note, artifact, import, reindex, and refinement calls can run up to one hour. Sync-generated pages, packets, and exports can run up to ten minutes.

## Reset Recovery

Use reset when the daemon-owned knowledge space has bad historical ingest, links, generated pages, or review state that should not be preserved.

Preview first:

```yaml
action: goodvibes.home_graph_reset
data:
  dry_run: true
```

Destructive reset requires typed confirmation:

```yaml
action: goodvibes.home_graph_reset
data:
  dry_run: false
  confirm: RESET
```

After reset:

1. Run `goodvibes.sync_home_graph`.
2. Reingest or relink manuals, product URLs, receipts, photos, and notes.
3. Run `goodvibes.home_graph_reindex`.
4. Refresh generated pages from the sidebar.
5. Retest Ask, Pages, and Map.

Do not manually delete SDK database rows. Do not use import as a reset substitute.
